# functional

Функциональная спецификация: Мобильное приложение «Smart To-Do Local»  
Системный аналитик: Кузьмин Артём Версия: 1.0  

Термин	     Определение
Задача	     Элемент списка дел, имеющий название, опциональные атрибуты (дата, время, категория, заметка, повторение, напоминание).  
Список	     Контейнер для задач (например, «Работа», «Личное»). Позволяет группировать задачи по контексту.  
Категория	  Метка, присваиваемая задаче для цветовой идентификации и группировки.  
Напоминание	Уведомление, которое приходит в заданное время до срока выполнения задачи.  

Список необходимых функций:  
1)	Управление задачами  
2)	Напоминания  
3)	Категории  
4)	Заметки  
5)	Повторение задач  
6)	Управление списками задач  
7)	Сортировка и группировка  
8)	Поиск и фильтрация  
9)	Сортировка и группировка  
10)	Вложения (файлы)  
11)	Срок выполнения (дата и время)
12)	Приоритет задач
13)	Подзадачи (Этапы)  

```
Функционал: Управление задачами  
Как пользователь  
Я хочу создавать, редактировать, удалять задачи и отмечать их выполнение  
Чтобы организовать свои дела.  

Сценарий: Создание задачи с обязательным названием  
Дано пользователь находится на главном экране  
И выбран активный список задач  
Когда пользователь нажимает кнопку "Добавить задачу"  
Тогда открывается экран создания задачи  
И курсор находится в поле "Название"  
Когда пользователь вводит название "Купить молоко"  
И заполняет остальные поля по желанию  
И нажимает кнопку "Сохранить"  
Тогда задача "Купить молоко" сохраняется в базе данных  
И отображается в списке задач  

Сценарий: Попытка создания задачи без названия  
Дано пользователь на экране создания задачи  
Когда поле "Название" оставлено пустым  
И пользователь нажимает кнопку "Сохранить"  
Тогда появляется сообщение "Введите название задачи"  
И сохранение не выполняется  

Сценарий: Отмена создания задачи  
Дано пользователь на экране создания задачи  
Когда пользователь нажимает кнопку "Отмена"  
Тогда приложение возвращается на главный экран  
И новая задача не создаётся  

Сценарий: Редактирование существующей задачи  
Дано на главном экране отображается задача "Купить молоко"  
Когда пользователь нажимает на эту задачу  
Тогда открывается экран редактирования с предзаполненными данными  
Когда пользователь изменяет название на "Купить хлеб"  
И нажимает "Сохранить"  
Тогда задача обновляется в базе данных  
И в списке отображается "Купить хлеб"  

Сценарий: Удаление задачи через свайп  
Дано на главном экране есть задача "Купить хлеб"  
Когда пользователь выполняет свайп влево по задаче  
И нажимает появившуюся кнопку "Удалить"  
Тогда задача "Купить хлеб" удаляется из списка  
И показывается уведомление с возможностью отменить удаление  

Сценарий: Удаление задачи через контекстное меню  
Дано на главном экране есть задача "Купить хлеб"  
Когда пользователь выполняет долгое нажатие на задаче  
И выбирает пункт "Удалить" в контекстном меню  
Тогда появляется диалог подтверждения удаления  
Когда пользователь подтверждает удаление  
Тогда задача "Купить хлеб" удаляется из списка  

Сценарий: Отметка задачи как выполненной  
Дано на главном экране отображается задача "Купить молоко"  
Когда пользователь нажимает на чекбокс рядом с задачей  
Тогда задача отмечается как выполненная  
И её отображение меняется (зачёркивание, полупрозрачность)  
И задача перемещается в конец списка или в раздел выполненных (согласно настройкам)  

Сценарий: Снятие отметки о выполнении  
Дано задача "Купить молоко" отмечена как выполненная  
Когда пользователь повторно нажимает на чекбокс  
Тогда отметка снимается  
И задача возвращается в обычное состояние

```

Как пользователь

Чтобы не пропустить важные дела.  

Сценарий: Установка напоминания при создании задачи

Сценарий: Отмена напоминания при редактировании

Когда пользователь открывает задачу на редактирование  
И убирает напоминание  
И сохраняет изменения  
Тогда запланированное уведомление отменяется  

Сценарий: Отказ в разрешении на уведомления  
Дано пользователь впервые пытается установить напоминание  
И система уведомлений запрещена для приложения  
Когда пользователь выбирает опцию "Напоминание"  
Тогда появляется диалог с предложением перейти в настройки для включения уведомлений  
И напоминание не устанавливается

Функционал: Повторение задач  
Как пользователь  
Я хочу создавать повторяющиеся задачи  
Чтобы автоматически планировать регулярные дела  

Сценарий: Создание ежедневной повторяющейся задачи  
Дано пользователь на экране создания задачи  
Когда пользователь выбирает тип повторения "Ежедневно"  
И указывает дату начала (сегодня)  
И сохраняет задачу  
Тогда задача создаётся с атрибутом повторения "Ежедневно"  
И на следующий день автоматически создаётся новая задача с теми же параметрами  

Сценарий: Повторение задачи с напоминанием  
Дано задача настроена на повторение "Еженедельно"  
И для неё установлено напоминание за час до дедлайна  
Когда наступает следующий срок выполнения  
Тогда создаётся новая задача  
И для неё также устанавливается напоминание за час до её дедлайна  


Сценарий: Завершение повторений по дате  
Дано задача с повторением "Ежемесячно" и датой окончания "31.12.2025"  
Когда после 31 декабря 2025 года наступает следующий срок  
Тогда новая задача не создаётся  

Сценарий: Отмена повторений после выполнения  
Дано существует повторяющаяся задача  
Когда пользователь отмечает текущий экземпляр задачи как выполненный  
Тогда новая задача на следующий период не создаётся

Сценарий: Создание новой категории при редактировании задачи  
Дано пользователь на экране создания задачи

Сценарий: Просмотр заметки  
Дано задача содержит заметку "Позвонить перед выходом"

Тогда открывается всплывающее окно с полным текстом заметки

Функционал: Управление списками задач

Я хочу создавать несколько списков задач

Сценарий: Создание нового списка

Сценарий: Удаление непустого списка

И после подтверждения задачи перемещаются, а исходный список удаляется

Когда пользователь открывает фильтр по категориям

Функционал: Вложения (файлы)  
Как пользователь  
Я хочу прикреплять файлы к задачам  
Чтобы хранить связанные документы, изображения и другие материалы вместе с задачей

Сценарий: Удаление вложения из задачи

Функционал: Срок выполнения (дата и время)  
Как пользователь  
Я хочу устанавливать дату и время выполнения для задач  
Чтобы планировать свои дела во времени и не пропускать дедлайны  

Сценарий: Установка даты выполнения при создании задачи  
Дано пользователь на экране создания задачи  
Когда пользователь выбирает поле "Срок"  
И выбирает дату "01.03.2026" в календаре  
И сохраняет задачу  
Тогда задача получает срок выполнения "01.03.2026"  
И в списке задач отображается дата "01.03.2026" рядом с задачей  

Сценарий: Установка даты и времени выполнения  
Дано пользователь на экране создания задачи  
Когда пользователь выбирает поле "Срок"  
И выбирает дату "01.03.2026"  
И включает опцию "Время"  
И устанавливает время "18:30"  
И сохраняет задачу  
Тогда задача получает срок "01.03.2026 18:30"  
И в списке задач отображается "01.03.2026, 18:30"  

Сценарий: Редактирование срока существующей задачи  
Дано существует задача "Сдать отчёт" со сроком "28.02.2026"  
Когда пользователь открывает задачу для редактирования  
И изменяет срок на "02.03.2026"  
И сохраняет изменения  
Тогда срок задачи обновляется на "02.03.2026"  

Сценарий: Удаление срока у задачи  
Дано задача имеет установленный срок "01.03.2026"  
Когда пользователь открывает задачу для редактирования  
И удаляет значение в поле "Срок" (оставляет пустым)  
И сохраняет изменения  
Тогда задача больше не имеет срока выполнения  
И дата не отображается в списке задач  

Сценарий: Сортировка задач по сроку выполнения  
Дано в списке есть задачи:  
  | Название		   | Срок         |  
  | Купить хлеб	   | 02.03.2026   |

  | Позвонить маме | (нет срока)  |  
Когда пользователь выбирает сортировку "По сроку (сначала ближайшие)"  
Тогда задачи отображаются в порядке:
